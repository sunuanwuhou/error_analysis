# 考试提分系统最新项目状态摘要

## 1. 当前阶段
当前阶段优先完成并优化系统本身，目标是先把系统主链路做稳、做顺手、做完整，再考虑大规模录入与更多扩展能力。

## 2. 当前项目基线
- 系统目标是服务提分，不是为了做系统而做系统
- 先整体后局部
- 先明确范围、边界、交互、数据规则、验收，再进入实现
- 已确认方案优先级高于优化建议
- 涉及页面交互、布局、入口、弹层、覆盖、跳转关系时，默认先给交互图
- 每次交付前必须自测
- 每形成阶段性结论后，同步更新项目文档

## 3. 已确认重点方案
### 3.1 画布功能 V1
- 入口：题目卡片右上角“画布”按钮，与“展开”同区域
- 打开方式：直接在题目内容区域上覆盖透明画布层
- 页面关系：不跳转页面
- 第一版工具：撤销 / 清空 / 退出
- 数据规则：每道题画布数据自动保存，再次打开自动恢复
- 风格要求：按钮与题目区域底色、风格统一

### 3.2 运行入口基线（本次新增）
- 当前在线实际运行入口：`v51_frontend`
- `frontend/src` 当前应视为主前端源码链路，不等于当前在线可见链路
- 后续凡涉及页面可见改动，默认优先修改 `v51_frontend`
- 若同时修改 `frontend/src`，必须明确说明其作用是“同步基线”还是“立即生效”

## 4. 当前问题结论
本次画布补丁出现“仓库有代码、构建也过了、页面看不到”的根因已经明确：

- 不是功能逻辑本身失败
- 不是补丁没有写进去
- 而是本次改动落在 `frontend/src`
- 当前实际运行页面却来自 `v51_frontend`
- 所以这是“目标链路错位”问题，不是“代码未生效”问题

## 5. 当前文档策略
当前建议长期维护以下 4 份文档：

1. 项目总协作规范（低频更新）
2. 已确认设计方案汇总（中频更新）
3. 最新项目状态摘要（高频更新）
4. 版本变更记录（追加更新）

## 6. 当前默认推进方式
- 先按项目层 → 模块层 → 功能层 → 细节层推进
- 优先在现有结构上增量修改，不轻易推翻旧页面
- 先做可用版本，再做扩展优化
- 优先交付可直接使用的结果
- 交付前做自测并记录结果
- 新增前端需求先确认“当前运行入口”和“本次修改入口”是否一致

## 7. 下一步执行基线
- 画布功能后续改造默认落在 `v51_frontend`
- 若需同步到 `frontend/src`，作为第二步，不替代对 `v51_frontend` 的修改
- 后续每次前端补丁都必须写清“运行入口 / 修改入口 / 验收入口”


## 9. 2026-04-03 画布 V2 体验收敛
- 入口回到题目卡片右上角
- 默认仅保留一个“画布”按钮
- 编辑态主工具收敛为：撤销 / 清空 / 退出
- 导出图片移入“更多”二级菜单
- 默认改为单色画笔
- 保存方式改为自动保存
- 状态文案收敛为：未标注 / 自动保存中 / 已保存
- 导出图片优先合成题目内容与画布笔迹
- 自动保存不再依赖整页 renderAll 刷新，避免编辑中断
