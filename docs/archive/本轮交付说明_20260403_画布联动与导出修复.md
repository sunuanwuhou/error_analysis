# 画布联动与导出修复说明 2026-04-03

## 本轮只修
- 画布保存后没有真正联动到题目数据
- 画布联动总览统计为 0
- 导出图片在部分环境下仍退化成空白/仅画布

## 根因
- 画布脚本读取错题数据时错误依赖 `window.errors`
- 当前系统真实数据源在主包闭包里的 `errors`，应通过全局函数 `findErrorById / getErrorEntries` 访问
- 因此保存链路实际找不到题目对象，导致保存未落库、联动总览为 0
- 导出链路里 `foreignObject` 在部分环境下会“技术上成功但内容仍近似空白”

## 本轮修复点
- 保存/联动改为优先使用 `findErrorById` 和 `getErrorEntries`
- 保存失败时明确提示“未找到题目数据”
- 画布联动总览打开时按真实错题数据重新统计
- 导出增加“空白结果检测”，一旦检测到近似空白，自动回退到文字重绘版题目 + 画布

## 验收
1. 画一笔后等待自动保存，状态应收敛为“已保存”
2. 打开“画布联动总览”，已有画布题目/累计保存次数/累计笔迹不应再为 0
3. 导出图片应至少包含题目文字 + 画布笔迹
