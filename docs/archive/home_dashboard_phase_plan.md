# 学习首页分期方案

## 目标

把现在的“学习统计”升级成真正的首页行动页。

首页不再只是展示数据，而是回答两个问题：

1. 我今天应该先做什么
2. 为什么先做这个

第一期先把首页做成可用的行动入口，后续再逐步增强自动判因和推荐能力。

## V1

### 目标

把首页从“统计展示”变成“行动入口”，并建立系统的常驻首页与工作台双层结构。

### 已落地范围

1. 系统进入后默认进入常驻学习首页，不再依赖弹窗
2. 首页第一屏优先展示“今日最该先做的动作”
3. 首页提供四个快捷入口：
   - 开始今日任务
   - 继续录题
   - 查看错题
   - 回看笔记
4. 首页保留并前置“今日任务”模块
5. 首页保留并前置“今日必须回看的笔记”模块
6. 首页保留“自动判因”的第一期基础版
7. 首页保留“顽固弱点榜”
8. 侧边栏“更多”中的 `统计` 入口改为 `首页`
9. 首页和错题工作台拆成两个一级空间：
   - `学习首页`
   - `错题工作台`
10. 首页按钮已经可以把用户分流到工作台

### 数据来源

第一期尽量复用现有数据，不新增复杂后端规则：

1. 今日任务
   - 来自 `buildPracticeTaskPack()`
   - 使用 `dailyQueue / reviewQueue / retrainQueue`
2. 必看笔记
   - 来自 dashboard 中 `mustReviewNote` 判定
   - 当前规则：认知问题或混合问题，且风险较高/复发/顽固
3. 自动判因
   - 复用 `buildDashboardWeakClusters()` 和 `inferClusterProblemType()`
   - 第一阶段只做：
     - 认知问题
     - 执行问题
     - 混合问题
     - 待观察
4. 顽固弱点榜
   - 复用现有 cluster 排序
   - 优先显示复发、高风险、顽固项

### 产品定位

第一期首页是“行动入口”，不是完整学习中台。

优先级顺序：

1. 先行动
2. 再解释
3. 最后看趋势

因此首页结构应保持：

1. 顶部行动建议
2. 今日任务
3. 必看笔记
4. 自动判因
5. 顽固弱点
6. 进度与趋势

## V2

V2 目标：让首页“更聪明”，同时把错题状态流转真正做实。

### 当前已落地

1. 新增并归一化正式阶段字段：
   - `captured`
   - `diagnosing`
   - `review_ready`
   - `retrain_due`
   - `mastered`
2. 新增并归一化判因字段：
   - `problemType`
   - `nextActionType`
   - `confidence`
   - `tip`
3. 导入、导出、CC 模板已经兼容这些字段
4. 错题卡片已经展示：
   - 阶段
   - 问题类型
   - 建议动作
   - 把握度
5. 错题卡片支持手动切换阶段，作为工作流推进入口

### 规划内容

1. 建立正式的错题状态流转：
   - `captured`
   - `diagnosing`
   - `review_ready`
   - `retrain_due`
   - `mastered`
2. 自动判因从基础统计升级为可解释规则面板
3. 顽固弱点支持更细的复发解释
4. 必看笔记支持更明确的推荐原因
5. 首页生成更稳定的“今日一句话建议”

### 建议规则

1. 认知问题
   - 同类题持续错
   - 错因指向概念/模型/知识点
   - 已有笔记但仍反复不会
2. 执行问题
   - 会做但超时
   - 粗心、漏条件、比较不全
   - 低把握、改答案、稳定性差
3. 混合问题
   - 既错又慢
   - 同时存在认知和执行信号
4. 必看笔记
   - 与今日任务直接关联
   - 且属于认知问题或混合问题

## V3

V3 目标：让任务包和首页完全按状态流转和判因结构工作。

### 当前已落地

1. 今日任务包已经按阶段拆分：
   - `待判因`
   - `待复盘`
   - `待复训`
2. 首页 hero 文案已经优先使用这三类阶段任务
3. 工作台新增任务视角筛选：
   - 全部
   - 待判因
   - 待复盘
   - 待复训
4. 首页“先补待判因”按钮已能直接带着对应任务视角进入工作台

### 规划内容

1. 今日任务包按状态生成：
   - 待复盘
   - 待复训
   - 必看笔记
2. 自动生成任务推荐顺序
3. 支持更强的高危弱点识别
4. 支持基于知识点和错因的任务包组合
5. 支持“先回笔记再做训练”的一键联动

### 期望结果

用户打开系统后，不需要自己思考路径，首页直接给出：

1. 现在先做什么
2. 为什么先做这个
3. 做完后下一步去哪

## V4

V4 目标：让系统从“记录错题”升级成“调度掌握进程”。

### 规划内容

1. 顽固弱点榜按复发、高风险、复训失败统一排序
2. 必看笔记支持推荐原因和一键回看
3. 训练结果能推动题目自动从 `review_ready / retrain_due` 流向 `mastered`
4. 首页趋势、任务、弱点、笔记形成闭环

## 导入 JSON 约定

旧 JSON 继续兼容，但后续标准导入格式建议补齐三层信息：

1. 题目数据
   - `type`
   - `subtype`
   - `subSubtype`
   - `question`
   - `options`
   - `answer`
   - `myAnswer`
   - `imgData`
   - `analysisImgData`
   - `optionImages`
2. 判因数据
   - `problemType`
   - `rootReason`
   - `errorReason`
   - `analysis`
   - `tip`
   - `actualDurationSec`
   - `targetDurationSec`
   - `confidence`
   - `nextActionType`
3. 状态数据
   - `workflowStage`
   - `status`
   - `masteryLevel`

### 推荐枚举

1. `problemType`
   - `cognition`
   - `execution`
   - `mixed`
   - `unknown`
2. `workflowStage`
   - `captured`
   - `diagnosing`
   - `review_ready`
   - `retrain_due`
   - `mastered`
3. `nextActionType`
   - `review_note`
   - `retrain`
   - `mixed_train`
   - `observe`

### 兼容策略

1. 旧 JSON 可以不带这些字段
2. 导入时系统自动补默认值和推断值
3. 后续从系统导出的 JSON 逐步采用新格式

## 设计原则

1. 首页优先给动作，不优先给报表
2. 规则优先可解释，不优先复杂
3. 先让数据流动起来，再追求算法更准
4. 错题、笔记、训练三条链路必须打通

## 当前执行顺序

当前按下面顺序推进：

1. 继续补 `V2` 的判因可编辑入口和规则解释
2. 继续补 `V3` 的首页到工作台任务跳转闭环
3. 开始做 `V4` 的必看笔记与顽固弱点联动
4. 最后再收口趋势与自动推荐文案
