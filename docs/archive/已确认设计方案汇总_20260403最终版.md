# 考试提分系统已确认设计方案汇总

## 1. 项目总方向

### 1.1 当前阶段目标
当前阶段优先完成并优化系统本身；系统做好后，再继续大规模录入笔记，再进入真正提分阶段。

### 1.2 项目根本目标
系统只是辅助工具，核心目标始终是考试提分。后续建议与改造应优先服务提分效率，而不是为了系统而系统。

### 1.3 推进方式
以后围绕项目时，优先给出完整方案、明确规则/边界/验收、按模块推进、每步自测并交付可用结果。

## 2. 画布功能已确认方案（V1 基线）

以下是已确认的画布 V1 方案，后续默认按此执行，除非用户明确要求修改。

### 2.1 入口位置
- 在题目卡片右上角增加“画布”按钮
- 与“展开”处于同一区域
- 不单独跳新页面

### 2.2 打开方式
- 点击“画布”后，直接在题目内容区域上覆盖透明画布层
- 采用原地覆盖，不跳转页面

### 2.3 风格要求
- 按钮与题目区域底色、风格保持统一
- 整体视觉尽量贴近现有页面，不做突兀新风格

### 2.4 第一版工具范围
V1 只保留以下工具：

- 撤销
- 清空
- 退出

本阶段不默认扩展更多画笔工具。

### 2.5 数据规则
- 每道题的画布数据自动保存
- 再次打开同一道题时自动恢复之前内容

### 2.6 默认要求
- 第一版以稳定、顺手、可恢复为优先
- 后续新增工具或能力，必须基于此方案迭代

## 3. 运行入口识别与改造目标基线（新增固化）

这是本次新确认的关键规则，后续默认执行。

### 3.1 问题背景
本次已确认：

- 当前在线实际运行入口是 `v51_frontend`
- 之前补丁主要改的是 `frontend/src`
- 因此出现“代码在仓库里、build 也过了、但页面看不到”的问题

问题本质不是代码没吸收，而是：

- 改动目标链路错了
- 实际运行壳与改动壳不一致
- 验收对象和修改对象没有对齐

### 3.2 后续默认规则
以后凡涉及页面功能、样式、按钮、入口、交互改造，默认先确认下面 4 件事：

1. 当前线上正在运行的入口目录是哪个
2. 用户当前看到的页面由哪个前端壳实际渲染
3. 本次修改目标是“主前端源码”还是“当前运行壳”
4. 验收页面到底对应哪个构建产物

### 3.3 当前已确认运行入口
当前阶段默认以：

- `v51_frontend`

作为“当前正在运行的页面入口”基线。

若后续切换运行入口，必须同步更新项目状态文档和版本变更记录。

### 3.4 改造优先级规则
以后默认优先顺序为：

1. 先改当前正在运行的入口
2. 再考虑是否同步回灌到 `frontend/src`
3. 若两套前端并存，必须明确说明：
   - 本次改的是哪套
   - 哪套会真正生效
   - 哪套只是源码基线或后备链路

- 当前运行入口：
- 本次修改入口：

- 页面验收入口：

### 3.6 验收新增要求
以后前端需求默认新增两条验收：

- 验证改动是否落在“当前运行入口”而不是只落在仓库源码里
- 验证用户当前在线页面是否能直接看到改动结果

## 4. 前端交互与布局类方案默认规则

凡涉及以下内容时，默认需要同步给出交互图或结构示意图：

- 页面布局调整
- 按钮位置变化
- 新入口增加
- 弹层 / 抽屉 / 覆盖层 / 跳转页
- 模块之间的流转关系
- PC 与移动端交互差异
- 原页面改造类需求

交互图至少要说明：

1. 入口位置
2. 触发后的页面变化
3. 是否跳转
4. 是弹层、覆盖层还是内嵌展开
5. 与原有功能的相对位置
6. 完成后的返回路径
7. 数据保存与恢复关系

## 5. 开发交付默认要求

### 5.1 优先交付可用结果
围绕系统项目时，应优先直接交付：

- 可用补丁
- 完整版本包
- 可直接替换文件
- 明确变更说明
- 自测结果说明

### 5.2 自测要求
交付前默认检查：

- 语法错误
- 重复声明
- 功能入口
- 样式对齐
- 移动端兼容
- 旧功能兼容
- 数据保存与恢复
- 运行入口是否命中
- 当前在线页面是否可见

### 5.3 对齐原则
功能、布局、按钮位置、风格，需要尽量与原有页面一致或与确认方案一致，不能只做“功能有了”但位置和样式偏掉。

## 7. 画布 V2 体验收敛方案（2026-04-03 新增）

### 7.1 目标
在保留画布主链路稳定的前提下，收敛首版按钮和颜色，降低刷题时的操作负担。

### 7.2 已确认内容
- 入口位置：题目卡片右上角
- 默认入口：仅保留一个“画布”按钮
- 打开方式：题目内容区域原地透明覆盖，不跳转
- 主工具：撤销 / 清空 / 退出
- 导出：移出主工具条，放入“更多”二级菜单
- 画笔：默认单色，不再首屏提供多颜色切换
- 保存方式：自动保存
- 状态文案：未标注 / 自动保存中 / 已保存
- 导出图片：优先导出“题目 + 画布”，失败时回退为仅画布

### 7.3 经验固化
第一版没有直接做成极简体验，不是因为方案不好，而是当时优先保链路闭环：能画、能存、能恢复、不卡。
后续默认经验：
- 首版先保行为闭环
- 闭环稳定后，再收敛按钮与交互
- 自动保存优先于强调手动保存
- 低频功能移出主工具条
