# 新版 / 旧版截图问题汇总（2026-04-10 重抓版）

本清单基于 2026-04-10 重新执行截图脚本后的结果整理。

- 抓取命令：`cd ui && npm run capture:parity`
- 截图目录：`artifacts/screenshot-parity/`
- 说明：本清单只记录本轮重抓后仍然存在的问题，不包含修复方案。

## 本轮结论

和 2026-04-09 那轮相比，这次 `/next` 已经明显前进：

- 移动端多数页面不再只是首页壳层，已经能进入对应功能页。
- 练习流已经能抓到实际题目作答态，不再只是说明页。
- 备份、搜索、错误工作台等页面已经出现更接近真实内容的新版截图。

当前剩余问题已经从“很多页面没进功能页”，收敛为“部分移动端布局异常”和“少数工具页仍未达到旧版工作形态”。

## P1

### 1. 首页整体信息层级混乱，首屏焦点不如旧版明确
- 现象：新版首页首屏同时堆叠大标题、说明文案、统计卡、侧边按钮和状态信息，视觉重心不稳定。
- 对比：旧版首页虽然也有多个模块，但“进入工作台”和“继续录题”更集中，首屏任务更直接。
- 影响：用户一进首页时，不容易立刻判断“现在该点哪里开始”。

### 3. 移动端错误工作台标题区布局异常，标题被竖排挤压，筛选区首屏可用性明显下降

- 对比：旧版移动端首屏是正常横向标题和可读筛选区；新版首屏标题区占用过多垂直空间。
- 影响：移动端首屏可读性和筛选操作效率明显下降。

## 本轮已明显改善

以下问题在本轮重抓中已不再是主要问题，因此从主问题清单中移除：

- 移动端大量页面停留在壳层首页，没有进入目标功能页
- 练习流只停留在说明页，没有进入作答态
- 备份、搜索、错误工作台等页面新版没有抓到实际内容

## 汇总结论

- 当前最明显的问题已经不是“功能页没打开”，而是“部分移动端布局已经打开但排版仍然不对”。
- 当前最难说已对齐的仍然是：
  - 移动端工作台布局
  - 笔记独立工具页语义
  - 过程图 / 画布的真实编辑工作区
- 如果只看这轮新截图，`/next` 已经比 2026-04-09 那轮接近可比对状态，但还不能说截图层面完全对齐旧版。
